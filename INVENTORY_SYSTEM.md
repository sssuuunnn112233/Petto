# 库存系统说明

## 功能概述

实现了一个完整的库存管理系统，让用户可以：

1. **从game-record页面购买物品** - 支持选择购买数量，物品会添加到库存中
2. **从pet页面使用库存物品** - 可以选择数量进行喂食/玩耍

## 系统架构

### 数据存储
- 库存数据存储在 `wx.getStorageSync('userInventory')` 中
- 格式：`{itemId: quantity}` 例如：`{"food_1": 3, "toy_2": 1}`

### 页面模式

#### 1. 普通购买模式（从game-record跳转）
- URL: `/pages/points-store/points-store`
- 功能：购买物品并添加到库存
- 按钮文本：购买/免费获取
- **新增功能**：付费商品支持数量选择（最多10个）
- 购买后：物品数量添加到库存

#### 2. 宠物使用模式（从pet页面跳转）
- URL: `/pages/points-store/points-store?category=food&from=pet`
- 功能：使用库存物品或购买新物品
- 显示：库存区域 + 商店区域
- 逻辑：优先使用库存，库存不足时可购买

### 核心方法

#### 库存管理
- `loadInventory()` - 加载库存数据
- `saveInventory()` - 保存库存数据
- `addToInventory(itemId, quantity)` - 添加物品到库存
- `removeFromInventory(itemId, quantity)` - 从库存移除物品
- `updateInventoryItems()` - 更新当前分类的库存物品列表

#### 购买逻辑
- `onBuyItem()` - 处理物品点击事件
- `showInventorySelector()` - 显示库存使用选择器
- `showQuantitySelector()` - 显示购买数量选择器（宠物模式）
- `showPurchaseQuantitySelector()` - 显示购买数量选择器（普通模式）
- `purchaseItem()` - 执行单个购买逻辑
- `purchaseMultipleItems()` - 执行多个购买逻辑

#### 使用逻辑
- `confirmInventoryUse()` - 确认使用库存物品
- `useInventoryItem()` - 执行库存物品使用
- `applyInventoryFeedingEffect()` - 应用库存喂食效果

## 用户体验流程

### 购买流程（game-record → 积分商店）
1. 用户从游戏记录页面点击"积分商店"
2. 进入普通购买模式
3. 选择物品：
   - **免费物品**：直接确认获取
   - **付费物品**：显示数量选择器
4. 选择购买数量（1-10个，受积分限制）
5. 确认购买，扣除积分，物品添加到库存
6. 显示"购买成功"详细信息

### 使用流程（pet → 积分商店）
1. 用户从宠物页面点击"喂食"
2. 进入宠物使用模式，显示食物分类
3. 如果有库存：
   - 显示库存区域，点击"使用"
   - 选择使用数量
   - 确认使用，从库存扣除，应用到宠物
4. 如果无库存：
   - 点击商店物品"喂食"
   - 选择购买数量
   - 确认购买并直接喂食

## 界面特性

### 数量选择器
- **普通购买模式**：
  - 标题："选择购买数量"
  - 预览：显示总费用和剩余积分
  - 按钮："确认购买"
  - 最大数量：受积分限制，最多10个

- **宠物使用模式**：
  - 标题："选择喂食数量"
  - 预览：显示饥饿值变化
  - 按钮："确认喂食"
  - 最大数量：受饥饿值和积分限制

### 库存显示
- 库存区域仅在宠物模式下显示
- 显示物品图标、名称、数量
- 库存为0的物品不显示

### 库存徽章
- 商店物品右上角显示库存数量徽章
- 红色圆形徽章，显示数量

## 数据流

```
购买模式：积分 → 库存（支持批量购买）
使用模式：库存 → 宠物状态

完整流程：
game-record → 积分商店(批量购买) → 库存 → pet页面 → 积分商店(使用) → 宠物状态
```

## 技术实现

### 状态管理
- `inventory: {}` - 当前库存数据
- `inventoryItems: []` - 当前分类的库存物品列表
- `showInventoryModal: false` - 库存使用弹窗状态
- `showQuantityModal: false` - 数量选择弹窗状态
- `previewTotalCost: 0` - 预览总费用

### 页面判断
- `fromPage` 参数区分页面来源
- 根据来源显示不同的界面和逻辑

### 数量限制
- **普通购买**：最多10个，受积分限制
- **宠物喂食**：受饥饿值和积分双重限制
- **库存使用**：受库存数量和饥饿值限制

### 数据同步
- 库存变化时自动保存到本地存储
- 页面显示时重新加载库存数据
- 分类切换时更新库存物品列表
